## 営業メールルール

### トーン＆マナー

**基本姿勢:**
- BtoB建設業界向け：丁寧かつ簡潔
- 「協力会社」「パートナー」としてのポジショニング
- 押し売り感を出さず、まず相手の事業に敬意を示す
- 実績・資格を必ず含める

**文体:**
- 敬語は丁寧語（です・ます調）で統一
- 1通目は短く（読了1分以内、300〜400文字程度）
- 絵文字は使わない

### メール構成テンプレート

1. **挨拶 + 自己紹介** - 「ライフアート株式会社の齋藤と申します」
2. **相手企業への言及** - 「御社の〇〇事業を拝見し」
3. **提案** - 「内装工事の協力会社としてお力になれると考えご連絡しました」
4. **強み** - 一級技能士在籍、スピード対応、13年の実績
5. **CTA** - 「まずはご挨拶かねてお打ち合わせのお時間をいただけませんか」

## CRMスプレッドシート設計

### メインシート「リード一覧」（A〜T列、20列）

| 列 | 項目名 | 用途 | データソース |
|----|--------|------|-------------|
| A | 会社名 | 基本情報 | スクレイピング |
| B | 業種分類 | ゼネコン/不動産/設計事務所/ハウスメーカー | スクレイピング |
| C | 代表者名 | 基本情報 | スクレイピング |
| D | 所在地 | 基本情報 | スクレイピング |
| E | TEL | 連絡先 | スクレイピング |
| F | メールアドレス | 送信先 | スクレイピング |
| G | HP URL | 企業サイト | スクレイピング |
| H | 取得元 | スクレイピングソース | 自動記録 |
| I | 取得日 | スクレイピング実行日 | 自動記録 |
| J | メール件名 | AI生成 or 手動 | AI/手動 |
| K | メール本文 | AI生成 or 手動 | AI/手動 |
| L | ステータス | 送信管理 | 自動/手動 |
| M | 送信日 | メール送信日時 | 自動記録 |
| N | 反応 | 返信あり/開封のみ/なし | 手動入力 |
| O | 反応メモ | 返信内容の要約 | 手動入力 |
| P | 営業フェーズ | リード/アポ調整/商談/受注/失注/保留 | 手動更新 |
| Q | 次のアクション | やるべきこと | 手動/AI |
| R | フォロー予定日 | 次回連絡日 | 手動入力 |
| S | フォロー回数 | 0〜 | 自動カウント |
| T | 備考 | 自由メモ | 手動入力 |

### ステータス定義（L列）

| ステータス | 意味 |
|-----------|------|
| **未送信** | メールアドレス取得済み、文面未作成 |
| **下書き済** | AI/手動で文面作成済み、確認待ち |
| **送信許可** | 齋藤社長が確認・承認済み |
| **送信済み** | メール送信完了 |
| **エラー** | 送信失敗（アドレス不正等） |

### 営業フェーズ定義（P列）

| フェーズ | 意味 |
|---------|------|
| **リード** | 初期接触段階（メール送信前〜送信済み） |
| **アポ調整** | 商談日程の調整中 |
| **商談** | 商談実施中 |
| **受注** | 成約 |
| **失注** | 不成約 |
| **保留** | 時期未定で保留 |

### 送信履歴シート「送信履歴」（A〜I列、9列）

メール送信のたびに自動で1行追加される。次回メール送信時の判断材料として使用。

| 列 | 項目名 | 用途 |
|----|--------|------|
| A | 送信日時 | いつ送ったか |
| B | リード行番号 | リード一覧との紐付け |
| C | 会社名 | どの会社に送ったか |
| D | 宛名 | 誰あてに送ったか（○○様 / ご担当者様） |
| E | メールアドレス | 送信先 |
| F | 件名 | どんな件名で送ったか |
| G | 本文 | どんな内容を送ったか（全文保存） |
| H | 送信種別 | 初回 / フォロー |
| I | フォロー回数 | 何回目の送信か |

## 業務フロー

```
[スクレイピング] → [スプレッドシート] → [AI文面生成] → [確認UI] → [メール送信]
  Python              Google Sheets        Claude API       GAS Web App     Gmail
```

### Step 1: スクレイピングで企業リスト取得
- iタウンページ / Google Maps で一都三県の対象企業を検索
- 会社名、住所、TEL、HP URLを取得
- 各企業HPにアクセスしてメールアドレスを取得
- Google Sheets APIでスプレッドシートに書き込み

### Step 2: 営業メール文面生成
- ターゲット業種に応じたテンプレートを使用
- Claude APIでパーソナライズされた文面を生成（将来拡張）

### Step 3: 齋藤社長による確認・承認
- Web UIで下書きを確認
- 必要に応じて編集
- ステータスを「送信許可」に変更

### Step 4: メール送信
- 送信許可が出たリードに対してメール送信
- 送信元: info@life-art2011.com
- 送信後、ステータスを「送信済み」に更新

```

├── CLAUDE.md                    ← このファイル（ルールブック）
├── scraper/
│   ├── requirements.txt         ← Python依存パッケージ

│   ├── itown_scraper.py         ← iタウンページスクレイパー

│   ├── website_email_scraper.py ← 企業HPからメール取得
```
