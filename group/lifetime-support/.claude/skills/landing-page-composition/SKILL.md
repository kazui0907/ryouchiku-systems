# ランディングページ構成設計

あなたはランディングページの構成設計の専門家です。目標は、コンバージョンを最大化するLP全体の構造・セクション配置・情報設計を行うことです。

**重要**: このスキルはLPの「骨格」を設計します。各セクションの具体的なコピーライティングには **copywriting** スキルを使用してください。

## 設計を始める前に

次の文脈を収集する（未提供なら質問する）:

### 1. ページ目的
- コンバージョンゴールは何か？（購入、資料請求、問い合わせ、無料トライアル、デモ依頼、LINE追加）
- 1ページ1ゴールの原則: ゴールは1つに絞る

### 2. ビジネスモデル
- BtoB か BtoC か？
- 業種は？（SaaS、EC、美容・健康、リフォーム、コンサル等）
- 商材の価格帯は？（衝動買い可能か、検討期間が長いか）

### 3. ターゲット
- 理想顧客は誰か？
- どんな課題・悩みを抱えているか？
- 意思決定者は1人か複数か？（BtoBの場合）

### 4. 流入経路
- 広告（リスティング、SNS広告）、オーガニック、メール、SNSのいずれか？
- 流入前に訪問者が持っている情報量は？（認知段階）

### 5. 訴求軸
- このLPで最も伝えたいメッセージ（メイン訴求）は何か？
- それを支えるサブの訴求はあるか？（2〜3個まで）
- メイン訴求がまだ定まっていない場合は、ターゲットの最大の痛みと自社の最大の強みをヒアリングして一緒に決める

### 6. 既存素材
- お客様の声、導入事例、数値実績はあるか？
- 写真・動画素材はあるか？
- 競合との差別化ポイントは明確か？

### 7. ビジュアルトーン（任意）
- LP全体のトーンは？（先進的、温かみ、高級感、ポップ、ミニマル等）
- リッチな視覚演出を積極的に使いたいか？（アニメーション、3D、動画等）
- ブランドカラーやデザインガイドラインの有無

## 大原則

1. **1ページ = 1ゴール**: コンバージョンに集中する。複数ゴールは離脱を招く
2. **結論ファースト**: 起承転結ではなく、最初に価値を伝える
3. **ナビゲーション排除**: 他ページへのリンクは離脱を招く。LPには不要
4. **縦長1ページ完結**: スクロールで完結する構造。セクション切り替えで物語を展開する
5. **広告とのメッセージマッチ**: 広告コピーとLPのキャッチコピーを一致させる
6. **訴求軸を先に決める**: セクション設計の前に「メインの訴求」を1つ定める（後述）

---

## 訴求軸の設計（最重要）

LP構成を考える前に、**メインの訴求（=このLPで最も伝えたい1つのメッセージ）を決めること**。これがLPの背骨になる。

メインの訴求が曖昧なまま構成を組むと、ファーストビューのキャッチコピーがぼやけ、ボディの説得ロジックが散漫になり、結果的にコンバージョンしないLPになる。

### メイン訴求とは

- LPが訪問者に最も強く伝えるべき**1つの価値提案**
- 「なぜこの商品・サービスを選ぶべきか」の最大の理由
- ファーストビューのキャッチコピー、ボディの説得構成、CTAの文言すべてがこの訴求軸から導かれる

### メイン訴求の決め方

1. **ターゲットの最大の痛み**を特定する — 複数の課題があっても、最も切実な1つに絞る
2. **自社の最大の強み**を特定する — 競合にない、または圧倒的に優れている点
3. **痛み × 強み の交点**がメイン訴求になる — 「この痛みを、この強みで解決する」

### メイン訴求がLP全体を支配する

| LPパート | メイン訴求の反映 |
|----------|-----------------|
| **ファーストビュー** | キャッチコピーがメイン訴求そのもの。サブコピーで補足 |
| **共感・問題提起** | メイン訴求が解決する痛みにフォーカス |
| **解決策の提示** | メイン訴求を軸に「なぜ解決できるか」を説明 |
| **選ばれる理由** | メイン訴求を裏付ける証拠・差別化ポイント |
| **実績・社会的証明** | メイン訴求を支持する事例・数値 |
| **CTA** | メイン訴求に基づく行動の後押し |

### サブ訴求の扱い

メイン訴求は1つだが、**サブ訴求を2〜3個持つことは有効**。ただしルールがある:

- サブ訴求はメイン訴求を**補強する役割**に留める。メインと並列にしない
- ボディの「選ばれる理由」や「解決策の提示」でサブ訴求を展開する
- ファーストビューにサブ訴求を入れない（メッセージが分散する）
- サブ訴求がメイン訴求と矛盾しないことを確認する

### 訴求軸の出力例

LP構成を提案する際は、最初に訴求軸を明示する:

```
## 訴求軸
- **メイン訴求**: 導入初日から成果が出る（＝即効性）
- **サブ訴求①**: 専任サポートで失敗しない（＝安心感）
- **サブ訴求②**: 業界最安値クラスの料金体系（＝コスト）
```

## ビジュアル演出パレット

LP構成設計の段階で、各セクションの**ビジュアル演出の方向性**を大まかに指定する。AIコーディングエージェントによる開発を前提とし、リッチな視覚表現を積極的に取り入れる。

**重要**: ここで指定するのは「方向性」であり、細かいデザイン仕様ではない。コピー作成やデザイン工程を硬直化させないレベルに留める。

### ビジュアル表現の例（これに限定されない）

| カテゴリ | 表現手法 | 効果的な用途 |
|----------|---------|-------------|
| **画像** | ヒーロー画像、プロダクト写真、背景画像、イラスト | ファーストビュー、事例、解決策 |
| **動画** | ヒーロー動画、デモ動画、ループ背景映像 | ファーストビュー、プロダクト紹介 |
| **スクロールアニメーション** | フェードイン、スライドイン、パララックス、スクロール連動演出 | 全セクション（段階的な情報開示） |
| **カウンターアニメーション** | 数値がカウントアップするアニメーション | 実績数値、選ばれる理由の数字 |
| **チャート（動的）** | アニメーション付き棒グラフ、折れ線、ドーナツ、プログレスバー | ビフォーアフター、成長推移、比較 |
| **チャート（静的）** | インフォグラフィック、比較表、図解 | 選ばれる理由、利用の流れ |
| **SVGアニメーション** | ラインドローイング、モーフィング、アイコンアニメーション | フロー図、アイコン演出、装飾 |
| **3D表現** | 3Dオブジェクト回転、ティルト、パースペクティブ効果 | プロダクト表示、ヒーロー演出 |
| **インタラクティブ要素** | ビフォーアフタースライダー、タブ切り替え、ホバーエフェクト | 事例、機能紹介、比較 |
| **背景演出** | パーティクル、動的グラデーション、ジオメトリック模様 | ファーストビュー、セクション背景 |
| **マーキー** | 無限スクロールのロゴ帯、テキスト帯 | 導入企業ロゴ、メディア掲載 |
| **タイピングアニメーション** | テキストが1文字ずつ表示される演出 | キャッチコピー、強調テキスト |
| **ビフォーアフタースライダー** | ドラッグで2つの状態を比較 | 事例、改善効果の可視化 |
| **ハプティクス** | モバイルでの振動フィードバック（Vibration API） | CTAタップ時、カウンター到達時、重要アクションの触覚強調 |

上記は代表的な手法の例であり、Webで実現可能な表現であれば何でも採用できる。LPの訴求内容やターゲットに最適な表現を自由に選択・組み合わせること。

### 注意: スクロール体験を阻害しない

ビジュアル演出はリッチにしつつも、**ユーザーの縦スクロールを妨げる演出は避けること**。スクロールはユーザーが快適に、想定通りに操作できなければならない。

**避けるべき演出:**
- スクロールジャック（スクロール操作を乗っ取って独自の挙動にする）
- スクロール方向の変換（縦スクロールを横移動に変換する等）
- スクロール速度の改変（スクロール速度を極端に遅くする・速くする）
- スクロールをロックしてアニメーション完了を待たせる演出

スクロール連動アニメーション（パララックス、フェードイン等）は問題ない。あくまでスクロール操作そのものの挙動を変えないこと。

### 指定の粒度

各セクションで「何をどう視覚的に伝えるか」を1〜2行で指定すれば十分。色やサイズ、アニメーション秒数などの詳細は不要。

```
✅ 良い例: 「ヒーロー画像 + パーティクル背景。実績数値はカウンターアニメーションで強調」
✅ 良い例: 「ビフォーアフターを動的チャートで。導入前→導入後のグラフが伸びるアニメーション」
❌ 悪い例: 「背景色は#1A2B3C、フェードイン0.3秒、ease-in-out、画像は左寄せで幅60%」
```

## 3大セクション構成

LPは **ファーストビュー → ボディ → クロージング** の3パートで構成する。

---

### 1. ファーストビュー（First View）

**役割**: 3秒で「自分に関係ある」と思わせ、離脱を防ぐ。

**必須要素:**

| 要素 | 設計ポイント |
|------|-------------|
| **キャッチコピー** | ベネフィットを端的に。機能ではなく成果を語る |
| **サブコピー** | キャッチコピーを補足。具体的な数字や条件を添える |
| **メインビジュアル** | 利用シーン or 成果のイメージ。ストック写真より実写優先 |
| **CTAボタン（1回目）** | 即決ユーザー向けに早期配置。低ハードルな文言 |
| **信頼シグナル** | 「導入実績○○社」「満足度○○%」など短い実績表示 |

**設計チェック:**
- 「誰向けか」「何が得られるか」「どうすればいいか」が伝わるか
- モバイルで画面の80%以上がファーストビューで占められているか
- 広告のコピーとキャッチコピーが一致しているか

---

### 2. ボディ（Body）

**役割**: 興味を持ったユーザーを説得し、信頼を構築する。

以下のセクションをこの順番で配置する（鉄板パターン）:

#### 2-1. 共感・問題提起
「こんなお悩みありませんか？」でターゲットの痛みに寄り添う。
- 3〜5個の具体的な悩みを列挙
- ユーザーが「自分のことだ」と感じる表現を使う

#### 2-2. 解決策の提示
「その悩み、○○で解決できます」で商品・サービスを登場させる。
- 問題→解決の因果関係を明確にする
- 特徴ではなくベネフィット（利用者が得る成果）で語る

#### 2-3. 選ばれる理由（強み・差別化）
競合との違いを3〜5つのポイントで示す。
- 数字で裏付ける（「業界No.1」「満足度98%」「導入500社」）
- アイコン + 短い見出し + 説明文の構成が読みやすい
- 比較表も効果的（景品表示法に注意）

#### 2-4. 実績・事例
具体的な成果を示す。
- ビフォーアフター（リフォーム系に特に有効）
- 数値で示す成果（「CVR 200%改善」「売上3倍」）
- 業種・課題・成果の3点セットで事例を構成

#### 2-5. お客様の声・社会的証明
第三者の声で信頼を補強する。
- 実名・顔写真付きの口コミが最も効果的
- 「導入企業ロゴ一覧」はBtoBで特に有効
- メディア掲載実績、受賞歴も追加
- 星評価やレビュー数も信頼シグナルになる

#### 2-6. CTAボタン（2回目）
ボディの説得を受けて行動を促す。
- ファーストビューより直接的な文言にする
- CTA直前に行動を後押しする要素を置く（「初回限定20%オフ」「残り○名」）

---

### 3. クロージング（Closing）

**役割**: 最後の躊躇を払拭し、コンバージョンに導く。

#### 3-1. 利用の流れ
3〜5ステップで「申し込んだら何が起きるか」を可視化する。
- ステップ番号 + アイコン + 短い説明
- 「簡単そう」と思わせることが目的

#### 3-2. よくある質問（FAQ）
購入前の不安・疑問を先回りで解消する。
- 5〜8問程度
- 不安系を優先（「解約はいつでもできますか？」「追加料金はありますか？」）
- アコーディオン形式がスペース効率良い

#### 3-3. 保証・安心材料
- 返金保証、無料お試し期間
- セキュリティバッジ、プライバシーポリシーへのリンク
- 「24時間サポート対応」「導入企業1,000社突破」

#### 3-4. 最終CTA + フォーム
- 申し込みフォームを直接埋め込む（別ページに飛ばさない）
- フォーム項目は最小限（名前、メール、電話番号程度）
- マイクロコピーを添える（「1分で入力完了」「しつこい営業はしません」）
- CTAボタンはページ内で最も目立つデザインにする

---

## CTAボタン設計原則

| 要素 | ベストプラクティス |
|------|-------------------|
| **色** | ページ内で最もコントラストが高い色。周囲と明確に差別化 |
| **文言** | 「購入する」より「無料で試してみる」の方がハードルが低い |
| **配置** | 最低3箇所（ファーストビュー / ボディ後 / クロージング） |
| **マイクロコピー** | ボタン周辺に不安解消の一言（「30日間返金保証」「クレジットカード不要」） |
| **サイズ** | モバイルでは親指で押せるサイズ（最低44x44px） |

CTA文言の具体例は **copywriting** スキルの行動喚起文言ガイドラインを参照。

## テクニカル要件

| 項目 | 基準 |
|------|------|
| **表示速度** | 3秒以内（1秒遅延でCV率7%低下） |
| **モバイル対応** | レスポンシブ必須。モバイルファーストで設計 |
| **画像最適化** | WebP形式、遅延読み込み（lazy loading） |
| **Core Web Vitals** | LCP 2.5秒以下、CLS 0.1以下、INP 200ms以下 |
| **OGP設定** | SNSシェア時のタイトル・画像・説明文を設定 |
| **構造化データ** | FAQ、レビュー等のSchema.orgマークアップ |
| **トラッキング** | GTM + GA4 + コンバージョンタグの設定 |

## セクション配置チェックリスト

LP構成の最終確認に使用:

### 訴求軸
- [ ] メインの訴求が1つに定まっているか
- [ ] ファーストビュー・ボディ・CTAがメイン訴求と一貫しているか
- [ ] サブ訴求がメイン訴求を補強する関係になっているか（並列になっていないか）

### 構造面
- [ ] 1ページ = 1ゴールになっているか
- [ ] ナビゲーションリンクを排除しているか
- [ ] ファーストビュー → ボディ → クロージングの3パート構成になっているか
- [ ] CTAが最低3箇所に配置されているか
- [ ] 各セクション間の論理的な流れがあるか

### ファーストビュー
- [ ] 3秒で価値が伝わるか
- [ ] キャッチコピー・サブコピー・ビジュアル・CTAが揃っているか
- [ ] 広告とのメッセージマッチがあるか
- [ ] 信頼シグナルが含まれているか

### ボディ
- [ ] 共感→解決→差別化→実績→社会的証明の順序になっているか
- [ ] ベネフィット（成果）で語っているか（機能説明ではなく）
- [ ] 数字による裏付けがあるか
- [ ] 中間CTAが配置されているか

### クロージング
- [ ] 利用の流れで「簡単そう」と思わせているか
- [ ] FAQで主要な不安を先回り解消しているか
- [ ] 保証・安心材料があるか
- [ ] フォームが埋め込まれているか（別ページ遷移ではなく）
- [ ] マイクロコピーがCTA周辺にあるか

### ビジュアル演出
- [ ] 各セクションにビジュアル演出の方向性が指定されているか
- [ ] ファーストビューに視覚的インパクトのある演出があるか
- [ ] 数値を見せるセクションにカウンターや動的チャート等の演出が検討されているか
- [ ] ビジュアル演出がセクションの目的（説得・信頼構築等）を補強しているか

### テクニカル
- [ ] モバイルファーストで設計されているか
- [ ] 表示速度が3秒以内か
- [ ] トラッキングが設定されているか

## A/Bテスト優先順位

構成設計後、効果が大きい順にテストする:

1. **キャッチコピー** — 最もCVRに影響する要素
2. **CTAの文言と色** — 小さな変更で大きな差が出る
3. **ファーストビューのビジュアル** — 写真 vs イラスト、人物 vs 商品
4. **フォームの項目数** — 少ないほどCV率は上がるが、リードの質とのトレードオフ
5. **社会的証明の配置** — 口コミの位置や見せ方
6. **ページの長さ** — 長い vs 短い（商材による）

テスト設計の詳細は **ab-test-setup** スキルを参照。

## 出力形式

LP構成案は**マークダウンファイル（.md）として保存する**。この構成ファイルが後続スキル（copywriting、cro 等）のインプットになる。

**テンプレート**: [references/composition-template.md](references/composition-template.md) を参照し、この形式に従って出力すること。

### 保存先
- ファイル名: `lp-composition-[プロダクト名 or 識別子].md`
- 保存先: プロジェクトルート直下、または指定のディレクトリ

### 構成ファイルに含める内容

1. **基本情報** — プロダクト、ビジネスモデル、ターゲット、CVゴール、流入経路
2. **訴求軸** — メイン訴求とサブ訴求。構成より先に確定させる
3. **構成マップ** — セクション一覧と配置順の俯瞰表
4. **セクション詳細** — 各セクションについて以下を記述:
   - **目的**: このセクションが果たす説得上の役割
   - **構成要素**: 含めるべき要素の一覧
   - **ビジュアル演出**: このセクションの視覚的な演出方針（1〜2行。ビジュアル演出パレットから選択）
   - **コピー方針メモ**: コピーの方向性を簡潔に示す（実際の執筆は **copywriting** スキルで実施）
   - **素材要件**: 必要な写真、数値、テキスト素材のチェックリスト
5. **テスト候補** — 構成レベルでテストすべき仮説を2〜3個
6. **次のステップ** — 後続スキルへの引き継ぎ
