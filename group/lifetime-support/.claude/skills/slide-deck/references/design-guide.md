# スライドデザインガイド

PDF書き出し前提のスライド資料（1280x720px / 16:9）のデザイン指針。

---

## 1. レイアウト

### 余白

- 各辺に **最低10%** のマージンを確保する（1280x720なら左右128px・上下72px以上）
- Tailwindでは `p-16`（64px）を最低ラインとし、`p-20`（80px）を推奨
- スライド面積の **15-20%以上を空白** にする。空白は「何もない」ではなく「読みやすさ」

### グリッドと整列

- 全要素をグリッドに揃える。数pxのズレでも違和感が生じる
- Tailwindの `gap`、`items-center`、`justify-between` で整列を保証する
- 同種の要素（カード、アイコン、箇条書き）は同一サイズ・同一間隔にする

### 視線の流れ

- Z型パターン: 左上 → 右上 → 左下 → 右下
- 最も重要な情報（タイトル、キーメッセージ）を左上または中央に配置
- レイアウトパターンはデッキ全体で統一する

## 2. タイポグラフィ

### フォントサイズ（Tailwindクラス対応）

| 要素 | 推奨 | Tailwind | 最低 |
|------|------|----------|------|
| メインタイトル | 48-60px | `text-5xl` 〜 `text-6xl` | `text-4xl`（36px） |
| サブタイトル / セクション見出し | 30-36px | `text-3xl` 〜 `text-4xl` | `text-2xl`（24px） |
| 本文 | 20-24px | `text-xl` 〜 `text-2xl` | `text-lg`（18px） |
| キャプション・脚注 | 14-16px | `text-sm` 〜 `text-base` | `text-xs`（12px） |

### 運用ルール

- **1スライドに使うサイズは3種類以内**（タイトル・サブ・本文）
- タイトルと本文のサイズ比は **1.5:1以上** の差をつける
- フォントは **最大2種類**（見出し用 + 本文用）。Geist Sans + Geist Mono で十分
- **行間**: `leading-relaxed`（1.625）を基本に。見出しは `leading-tight`（1.25）

### ウェイトの使い分け

| 用途 | Tailwind |
|------|----------|
| メインタイトル | `font-bold`（700）または `font-extrabold`（800） |
| サブタイトル | `font-semibold`（600）または `font-bold`（700） |
| 本文 | `font-normal`（400）または `font-medium`（500） |
| キャプション | `font-normal`（400）または `font-light`（300） |

## 3. 配色

### 基本ルール

- 色は **Tailwind のカラーパレット** を使う。`#3b82f6` のような生のHEX値ではなく `blue-500` のようなクラスで指定する
- 使用色は **3-4色** に制限する
- 配色比率: **ベース70% / メイン25% / アクセント5%**

### 色の役割

| 種類 | 役割 | 例 |
|------|------|-----|
| ベースカラー | 背景・地色 | 白、`gray-50`、`gray-100` |
| メインカラー | 見出し・キー要素 | ブランドカラー、`blue-600`、`gray-900` |
| アクセントカラー | 強調・注目喚起 | `amber-500`、`emerald-500`（面積を絞る） |
| テキストカラー | 本文・説明 | `gray-600` 〜 `gray-700`（真っ黒は避ける） |

### コントラスト

- 文字と背景のコントラスト比 **4.5:1 以上**
- 本文に `#000000`（真っ黒）は使わない → `gray-800`（#1f2937）や `gray-900`（#111827）を使う
- 補足テキストは `gray-500` 〜 `gray-600`
- ダーク背景の場合、本文は `white` ではなく `gray-200` 〜 `gray-300` で柔らかくする

### アクセシビリティ

- 色だけで情報を伝えない。ラベル・パターン・アイコンを併用する
- 赤/緑のみの対比を避ける（色覚異常者の約8%が識別困難）
- チャートでは色に加えてラベルを直接配置する

## 4. 情報構造

### 1スライド1メッセージ

- 1枚に伝えることは **1つだけ**
- スライド枚数が増えることを恐れない。分かりやすさが最優先
- スライド上部にタイトルとして「このスライドで伝えたいこと」を明示する

### 3秒ルール

- 読み手が **3秒以内** にスライドの主旨を理解できること
- タイトル → ビジュアル → 補足テキスト の順に視線が流れるように設計する

### テキスト量

- 1スライドのテキスト量は **100-150文字程度** に制限する
- 長い段落は載せない。要点のみ

### 箇条書き

- 最大 **6項目**（理想は3-5項目）
- 1項目 **20-30文字** まで
- 階層は **最大2段** まで
- 全項目にアイコンを付ける必要はない（過剰使用は逆効果）
- 箇条書き間に十分な余白を確保する（`gap-4` 以上）

## 5. ビジュアル要素

### アイコン（react-icons）

- 1デッキ内で **1スタイルに統一**（outline か filled のどちらか）
- 同じ文脈のアイコンは **全て同じサイズ** にする（`size={24}` や `size={32}` で統一）
- 推奨セット: `react-icons/fi`（Feather Icons）— outline で統一感が出やすい
- アイコンの背景に丸角ボックス（`rounded-xl p-3 bg-blue-50`）を付けるとモダンな印象に

### チャート（recharts）

- 1スライドにチャートは **1-2個** まで
- シンプルさを最優先。装飾的なグリッド線・目盛りは最小限に
- データラベルを直接チャートに配置し、凡例を別置きしない
- カテゴリの色数は **6色以内**
- チャートタイプの選び方:
  - 比較 → 棒グラフ
  - 推移 → 折れ線グラフ
  - 構成比 → 円グラフ（5項目以下のみ）
  - 相関 → 散布図

### QRコード（qrcode.react）

- サイズは `128` 〜 `160` が読み取りやすい
- 周囲に余白を確保する
- スライドの端や補足的な位置に配置する

## 6. リッチデザイン

スライドは「テキストを並べた資料」ではなく「視覚的に設計されたプレゼンテーション」として作る。全てのスライドにビジュアル要素を入れ、テキストだけのスライドは作らない。

### デザイン思考

スライドを作る前に、明確な美的方向性を決める:

- **トーン**: 対象に合った雰囲気を選ぶ。洗練・モダン、テック・クール、温かみ・親しみやすさ、大胆・インパクト重視など
- **差別化**: 何がこのデッキを印象的にするか？記憶に残る一要素を意識する
- **一貫性**: 選んだ方向性をデッキ全体で貫く。色・レイアウト・装飾のトーンを統一する

### レイアウトの多様性

同じレイアウトの繰り返しは単調になる。以下のパターンを組み合わせてリズムを作る:

| パターン | 用途 | 構成 |
|----------|------|------|
| **ヒーロー中央** | タイトル、キーメッセージ | 大きなテキスト＋アイコンを中央配置 |
| **カードグリッド** | 複数項目の並列比較 | 2〜4枚のカードを `grid` で配置 |
| **左右分割** | テキスト＋ビジュアル | `grid-cols-2` で左にテキスト、右に図解やチャート |
| **統計ヒーロー** | 数値のインパクト | `text-7xl font-extrabold` の大きな数字＋ラベル |
| **タイムライン/フロー** | プロセス、ステップ | 横並びのステップ＋矢印で接続 |
| **リスト行** | 項目の詳細一覧 | 各行にアイコン＋タイトル＋説明＋補足を横並び |

**リズムの作り方:**
- レイアウトパターンを2-3種類組み合わせて視覚的なメリハリを作る
- 情報密度の高いスライドと低いスライドを交互に配置する
- 重要なメッセージほど余白を多く取り、情報密度を意図的に下げる

### ビジュアル要素の活用

テキストだけのスライドは禁止。必ず以下のいずれかを含める:

- **アイコン（背景付き）**: `<div className="w-12 h-12 rounded-xl bg-blue-100 flex items-center justify-center"><FiIcon className="text-blue-600" size={24} /></div>`
- **カラーカード**: `bg-gray-50 rounded-2xl p-6` のカードに情報をまとめる。カードごとに異なるアクセントカラーを使う
- **ヒーロー数字**: `<span className="text-7xl font-extrabold text-blue-600">98%</span>` — 数値データは大きく見せる
- **ピル型バッジ**: `<span className="text-sm font-medium text-emerald-600 bg-emerald-50 px-3 py-1 rounded-full">推奨</span>`
- **コードブロック**: `bg-gray-900 rounded-xl px-6 py-4 font-mono` — 技術的な内容はコードブロックで
- **チャート**: recharts で棒グラフ、折れ線、円グラフなどデータを可視化
- **図解・フロー**: flexbox でステップやプロセスを視覚的に表現

### 背景と装飾

ソリッドカラーだけでなく、背景にも工夫を入れる:

- **グラデーション背景**: `bg-gradient-to-br from-blue-600 to-indigo-800` — セクションタイトルやクロージングに
- **薄いパターン背景**: 白背景の上に `bg-gray-50` のカードを置いて奥行きを出す
- **アクセントライン**: `<div className="w-20 h-1 bg-amber-500 rounded-full mb-6" />` — タイトルの下に色付きのラインを引く
- **ボーダーアクセント**: `border-l-4 border-blue-500 pl-6` — 引用や強調ブロックの左ボーダー
- **カラー背景スライド**: `bg-blue-50` や `bg-amber-50` — セクション区切りにブランドカラーの淡い背景を使う

### タイポグラフィの演出

フォントは Geist Sans / Geist Mono の2種だが、サイズ・ウェイト・色で豊かな表現ができる:

- **超大文字**: キーメッセージは `text-6xl font-extrabold tracking-tight` で圧倒的な存在感を出す
- **サイズコントラスト**: タイトルと本文の差を大きくする（`text-5xl` と `text-lg` など）。差が大きいほど洗練された印象になる
- **色で階層**: タイトル `text-gray-900`、本文 `text-gray-600`、補足 `text-gray-400` と明度で情報の重要度を表現
- **モノスペースの活用**: 技術的な用語、数値、コマンドは `font-mono` で差別化する
- **レタースペーシング**: 大見出しに `tracking-tight`、キャプションに `tracking-wide uppercase text-xs` で変化をつける

### 色の戦略

3-4色の制限の中で最大限の豊かさを出す:

- **支配色＋鋭いアクセント**: ベースカラーを広く使い、アクセントカラーは面積を極限まで絞る。小さいほど目立つ
- **カードごとの色分け**: 並列する要素には `blue-100`、`amber-100`、`emerald-100` など淡い色相違いの背景を割り当てる
- **背景色の変化**: 白背景と淡いカラー背景（`blue-50`、`gray-50` など）を交互に使い、視覚的リズムを作る
- **意味のある色**: 肯定的＝emerald、警告＝amber、否定的＝red、情報＝blue など、色に意味を持たせる

## 7. アンチパターン

### やってはいけないこと

| カテゴリ | NG | 理由 |
|----------|-----|------|
| テキスト | 18px未満のフォント | PDF拡大しないと読めない |
| テキスト | フォント3種類以上 | 統一感がなくなる |
| テキスト | テキストの壁（150文字超） | 読む気を失う |
| テキスト | 見出しと本文のサイズ差がない | 階層が不明瞭になる |
| 色 | 5色以上の使用 | ごちゃごちゃする |
| 色 | 低コントラスト | 読みにくい |
| 色 | 本文に `#000000` | コントラストが強すぎて目が疲れる |
| 色 | 赤/緑のみの対比 | 色覚異常者に伝わらない |
| レイアウト | 余白なし | 窮屈で読みにくい |
| レイアウト | 要素の不揃い | 無秩序な印象 |
| レイアウト | スライド毎にレイアウトが変わる | 一貫性がなく信頼感が低下 |
| 情報 | 1スライドに複数メッセージ | 何を覚えるべきか分からない |
| 情報 | 箇条書き7項目以上 | 情報過多で処理できない |
